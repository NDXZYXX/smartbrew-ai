### 2.1 device（设备表）**`V1`**

| 属性 | 内容 |
|------|------|
| **对应功能** | F003 WiFi联网、F004 MQTT上传、F007 心跳上报、F011 设备注册、F020 实时监控 |
| **存储引擎** | InnoDB |
| **说明** | ESP32设备注册信息。在线状态通过心跳机制（F007）动态更新，Redis中同时缓存实时状态供快速查询。 |

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|:--:|--------|------|
| id | BIGINT UNSIGNED | 是 | AUTO_INCREMENT | 主键 |
| device_id | VARCHAR(64) | 是 | - | 设备唯一标识（ESP32 MAC地址） |
| device_name | VARCHAR(100) | 是 | - | 设备名称，用户可自定义 |
| device_secret | VARCHAR(64) | 是 | - | 设备密钥，用于MQTT认证 |
| status | TINYINT | 是 | 0 | 在线状态：0-离线, 1-在线（由心跳超时机制自动置为离线） |
| ip_address | VARCHAR(45) | 否 | NULL | 最近一次上报的IP地址（来自F007心跳） |
| firmware_version | VARCHAR(20) | 否 | NULL | 当前固件版本号（来自F007心跳） |
| last_heartbeat_time | DATETIME | 否 | NULL | 最近一次心跳时间 |
| create_time | DATETIME | 是 | CURRENT_TIMESTAMP | 设备注册时间 |
| update_time | DATETIME | 是 | CURRENT_TIMESTAMP | 最后更新时间（ON UPDATE） |

**索引：**

| 索引名 | 类型 | 字段 | 说明 |
|--------|------|------|------|
| PRIMARY | 主键 | id | - |
| uk_device_id | UNIQUE | device_id | 设备ID全局唯一 |
| idx_status | 普通 | status | 按在线状态筛选设备 |

### 2.2 sensor_data（传感器数据表）**`V1`**

| 属性 | 内容 |
|------|------|
| **对应功能** | F001 温度采集、F002 环境温湿度、F004 MQTT上传、F012 数据接收、F013 数据存储、F014 实时接口、F015 历史接口、F020 监控面板、F021 历史曲线 |
| **存储引擎** | InnoDB |
| **说明** | 核心数据表，每30秒（F001温度）+ 每60秒（F002环境）合并为一条MQTT消息上传，统一写入此表。数据量大，建议按月分区。 |

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|:--:|--------|------|
| id | BIGINT UNSIGNED | 是 | AUTO_INCREMENT | 主键 |
| device_id | VARCHAR(64) | 是 | - | 设备编号 |
| tank_temperature | DECIMAL(5,2) | 是 | - | 桶内温度（℃），DS18B20，采集周期30s |
| env_temperature | DECIMAL(5,2) | 否 | NULL | 环境温度（℃），DHT22，采集周期60s |
| env_humidity | DECIMAL(5,2) | 否 | NULL | 环境湿度（%RH），DHT22，采集周期60s |
| create_time | DATETIME(3) | 是 | - | 采集时间戳（设备端生成，精度到毫秒） |

**索引：**

| 索引名 | 类型 | 字段 | 说明 |
|--------|------|------|------|
| PRIMARY | 主键 | id | - |
| idx_device_time | 普通 | (device_id, create_time) | 单设备历史查询（F015） |
| idx_create_time | 普通 | create_time | 全局时间范围查询 |

**分区建议：**
```sql
-- 按月RANGE分区，保留12个月数据
ALTER TABLE sensor_data PARTITION BY RANGE (TO_DAYS(create_time)) (
    PARTITION p202401 VALUES LESS THAN (TO_DAYS('2024-02-01')),
    ...
);
```

### 2.8 device_heartbeat（设备心跳表）**`V1`** ★新增

| 属性 | 内容 |
|------|------|
| **对应功能** | F007 设备心跳上报 |
| **存储引擎** | InnoDB |
| **说明** | ESP32每60秒上报一次心跳（RSSI、Heap内存、固件版本、IP），用于在线状态检测和设备健康监控。与device表分离，避免频繁更新device主表。 |

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|:--:|--------|------|
| id | BIGINT UNSIGNED | 是 | AUTO_INCREMENT | 主键 |
| device_id | VARCHAR(64) | 是 | - | 设备编号 |
| rssi | SMALLINT | 是 | - | WiFi信号强度（dBm），如 -45 |
| heap_free | INT UNSIGNED | 是 | - | ESP32可用堆内存（字节） |
| firmware_version | VARCHAR(20) | 是 | - | 当前固件版本 |
| ip_address | VARCHAR(45) | 是 | - | 设备当前IP |
| uptime_seconds | INT UNSIGNED | 否 | NULL | 设备运行时长（秒） |
| create_time | DATETIME | 是 | CURRENT_TIMESTAMP | 心跳时间 |

**索引：**

| 索引名 | 类型 | 字段 | 说明 |
|--------|------|------|------|
| PRIMARY | 主键 | id | - |
| idx_device_time | 普通 | (device_id, create_time) | 按设备查心跳历史 |
| idx_time | 普通 | create_time | 定时清理过期心跳数据 |

**维护策略：**
- 服务端定时任务：超过120秒（2个心跳周期）无心跳 → 将`device.status`置为0（离线）
- 心跳数据仅保留最近7天，定时清理
- Redis中维护 `device:online:{device_id}` 键，TTL=120s，每次心跳刷新TTL
