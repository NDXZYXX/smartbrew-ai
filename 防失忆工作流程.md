# Smart Brew AI — 防失忆工作流程（复制粘贴专用）

> 本文档包含所有场景的**可复制粘贴的 prompt 模板**。每次新开AI会话时，复制对应的提示词发给AI即可恢复上下文。

---

## 一、新会话启动（最重要）

每次新开 AI 会话时，复制下面这段发给 AI：

```
我正在开发 Smart Brew AI 智能发酵监控系统。

项目目录：C:\Users\Administrator\Desktop\酒桶发酵

请按以下步骤恢复上下文：
1. 先读取 PROJECT_ROADMAP.md，了解项目全貌和当前进度
2. 再读取 PROJECT_STATUS.md，了解昨天做了什么、有哪些未解决的问题
3. 最后告诉我你理解的当前状态，以及下一步应该做什么，我来确认
```

---

## 二、每日收工（当天结束）

当天开发结束、准备关掉 AI 前，发这段让 AI 写日志：

```
今天的开发结束了，请更新 PROJECT_STATUS.md：
1. 在"每日日志"末尾追加今天的日志（日期、完成事项、Git提交记录、遇到的问题、明日计划）
2. 更新 PROJECT_ROADMAP.md 中已完成子任务的复选框为 [x]
3. 如果某个 Phase 全部完成了，把 Phase 状态改为 ✅ 已完成，更新进度总览表格
4. 更新 PROJECT_STATUS.md 顶部"最后更新"时间
```

---

## 三、Phase 完成

某个 Phase 所有子任务完成后：

```
Phase N（XXX）已全部完成，请更新：
1. PROJECT_ROADMAP.md：该 Phase 状态改为 ✅ 已完成，进度总览填入完成日期
2. PROJECT_STATUS.md：总体进度条同步更新
3. 在 PROJECT_STATUS.md 今日日志中记录：Phase N 完成
```

---

## 四、部分子任务完成（开发中更新）

开发过程中完成了一些子任务但 Phase 还没完结：

```
我完成了以下子任务：
- Phase X 的 X.1（XXX）
- Phase X 的 X.2（XXX）

请更新 PROJECT_ROADMAP.md 中对应复选框为 [x]，并更新进度总览的进度数字（如 2/5）。
同时把当前进行中的 Phase 状态改为 🔄 进行中。
```

---

## 五、遇到阻塞问题

开发遇到问题被迫暂停时：

```
当前遇到阻塞问题：（描述问题）
Phase N 暂时无法继续。

请更新 PROJECT_STATUS.md：
1. "当前状态"→"阻塞问题" 记录具体问题
2. 本日日志中详细记录问题现象和已尝试的解决方案
```

---

## 六、问题解决恢复开发

之前阻塞的问题解决了：

```
之前的阻塞问题已解决：（简述如何解决的）

请更新 PROJECT_STATUS.md：
1. "当前状态"→"阻塞问题" 改为"无"
2. 本日日志记录解决过程
```

---

## 七、新增任务或需求变更

需求有变化或要加新功能时：

```
需要新增/变更以下内容：
（描述变更）

请同步更新：
1. 功能需求列表.md（如果是新功能需求）
2. 数据库列表.md（如果涉及数据库变更）
3. PROJECT_ROADMAP.md（如果涉及新Phase或子任务）
4. PROJECT_STATUS.md 本日日志记录变更
```

---

## 八、Git 提交

需要提交代码时：

```
请帮我创建 Git 提交：
- 查看 git status
- 查看 git diff
- 生成符合项目规范的 commit message（feat:/fix:/docs:/refactor:/chore:）
- 执行提交（不要 push）
```

## 十、状态标记规范

| 标记 | 含义 | 用于 |
|------|------|------|
| ⬜ | 未开始 | Phase 状态 |
| 🔄 | 进行中 | Phase 状态 |
| ✅ | 已完成 | Phase 状态 / 文档状态 |
| ⏸️ | 暂停 | Phase 状态 |
| `[ ]` | 未完成 | 子任务复选框 |
| `[x]` | 已完成 | 子任务复选框 |

---

## 十一、Git 提交类型

| 前缀 | 用途 | 示例 |
|------|------|------|
| `feat:` | 新功能 | `feat: DSP32温度采集与MQTT上报` |
| `fix:` | 修复Bug | `fix: 修复心跳超时判断逻辑` |
| `docs:` | 文档变更 | `docs: 更新数据库ER图` |
| `refactor:` | 重构 | `refactor: 提取MQTT公共模块` |
| `test:` | 测试 | `test: 添加告警规则单元测试` |
| `chore:` | 其他 | `chore: 配置Docker Compose` |
